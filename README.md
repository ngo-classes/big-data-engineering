# PySpark 3.5.1 Local Development Setup

This guide explains how to set up the local PySpark environment for the Big Data Engineering course.

The setup is designed to support both:

1. Running PySpark scripts directly with Python:

   ```bash
   python verify_spark.py
   ```

2. Running PySpark scripts through Spark's submit command:

   ```bash
   spark-submit --master "local[*]" verify_spark.py
   ```

The setup uses a Conda environment installed with `-p` so that the environment path is predictable and does not depend on a user's home directory. This is important for Windows users whose account name might contain spaces (like me!)

---

## Repository Layout

The repository should include the following files and directories:

```text
big-data-engineering/
├── environment.yml
├── verify_spark.py
├── setup-pyspark-windows.ps1
├── setup-pyspark-mac.sh
├── spark-conf/
│   └── spark-defaults.conf        # generated by setup script
└── hadoop/
    └── bin/
        ├── ...
        └── winutils.exe           # Windows only
```

The `spark-conf/spark-defaults.conf` file is generated by the setup scripts and contains machine-specific absolute paths. It is 
explicitly ignored by the local `.gitignore` inside `spark-conf`.

For both setup scenarios (Windows and Mac/Linux), it is assumed that you have cloned and are inside this repository. 

```bash
git config --global core.autocrlf false # run this if you are on Windows
git clone https://github.com/ngo-classes/big-data-engineering
cd big-data-engineering
```

## Windows Setup

### Create the Conda environment in Windows

Use a path without spaces. Do **not** create the environment under a user folder such as:

```text
C:\Users\First Last Name\...
```

Instead, use:

```powershell
conda env create -p C:\conda-envs\pyspark-3.5.1 -f environment.yml
```

Activate it:

```powershell
conda activate C:\conda-envs\pyspark-3.5.1
```

### Configure the Windows environment

From PowerShell, inside the repository:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
.\setup-pyspark-windows.ps1
```

The script configures the current PowerShell session. It does **not** run Python or `spark-submit`.

It sets:

```text
JAVA_HOME
SPARK_HOME
SPARK_CONF_DIR
HADOOP_HOME
PYSPARK_PYTHON
PYSPARK_DRIVER_PYTHON
TEMP
TMP
SPARK_LOCAL_DIRS
```

It also creates these Windows-safe temporary directories:

```text
C:\tmp
C:\spark-tmp
```

This avoids Spark failures caused by spaces in Windows user paths.

**You will need to rerun this script at the beginning of any session.**

## macOS Setup

### Create the Conda environment in MacOS

Create the environment in a predictable location:

```bash
conda env create -p "$HOME/conda-envs/pyspark-3.5.1" -f environment.yml
```

Activate it:

```bash
conda activate "$HOME/conda-envs/pyspark-3.5.1"
```

### Configure the macOS environment

From a terminal inside the repository:

```bash
source setup-pyspark-mac.sh
```

The script sets:

```text
JAVA_HOME
SPARK_HOME
SPARK_CONF_DIR
PYSPARK_PYTHON
PYSPARK_DRIVER_PYTHON
TMPDIR
SPARK_LOCAL_DIRS
```

It also creates a temporary Spark working directory under:

```text
/tmp/spark-tmp-$USER
```

## Run the verification script

Run this in the same terminal window:

```bash
python verify_spark.py
```

You should see output similar to:

```text
Spark Version: 3.5.1
+-------+
|   test|
+-------+
|Success|
+-------+
```

You may also test with `spark-submit`:

```bash
spark-submit --master "local[*]" verify_spark.py
```
